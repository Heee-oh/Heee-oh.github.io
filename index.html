Problem Solving

새 채팅방 첫 메시지 실시간 수신 문제 해결

구매자가 첫 메시지를 보내면 DB에는 저장되지만 판매자 화면에는 새 채팅방이 즉시 보이지 않는 문제가 있었습니다.

문제

수신자가 새 chatRoomId를 모름

기존 구조는 `/sub/chat/room/{chatRoomId}` 방 단위 구독만 사용했습니다. 새 방의 첫 메시지에서는 수신자가 아직 방 ID를 모르기 때문에 구독할 수 없었습니다.

해결

사용자 단위 이벤트 채널 추가

첫 메시지일 때만 `/sub/members/{memberId}/chat-events`로 이벤트를 발행해 수신자가 채팅방 목록을 다시 조회하고 새 방을 발견하도록 만들었습니다.

효과

중복 알림 없이 실시간성 확보

일반 메시지는 기존 방 단위 브로드캐스트를 유지하고, 새 방 발견에만 사용자 채널을 사용해 불필요한 재조회와 중복 알림을 줄였습니다.
