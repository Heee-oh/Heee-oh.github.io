01. Introduction

안정성, 성능, 무결성.
타협 없는 백엔드 개발자 홍길동입니다.

수많은 동시 요청 속에서도 한치의 데이터 오차도 허용하지 않는 정합성 보장 아키텍처 설계에 강한 자부심을 느낍니다.

단순 기능 구현을 완성이라 타협하지 않으며, 데이터 크기 증가에 대비하여 쿼리 실행 계획 분석(MySQL EXPLAIN)을 통해 인덱스를 효율적으로 재구축하고 성능 병목을 예방하는 것을 즐깁니다.

"지속 가능한 시스템을 지탱하는, 보이지 않는 곳의 철학을 고수합니다."

AI Concentration Analysis / Face Recognition

FLOW

My Role

분석 데이터 흐름과 예외 케이스 정리

Problem Solving

측정 데이터와 요약 결과의 책임 분리
Situation
얼굴 인식 결과가 시간 단위로 생성되면서 원본 분석값과 사용자에게 보여줄 요약 결과를 함께 관리해야 했습니다.
Task
측정 단위, 세션 단위, 사용자 단위 데이터의 책임을 분리하고 조회 API의 결과 기준을 명확히 해야 했습니다.
Action
측정 기록과 요약 결과를 분리해 저장하고, 조회 API에서는 세션 기준으로 집중도 결과를 조합하도록 구조화했습니다.
Result
분석 데이터 저장과 결과 조회 흐름을 분리해 기능 확장과 화면 연동이 쉬운 구조로 정리했습니다.

SERVICE FLOW

My Role

굿즈 등록/수정/조회 API 설계

교환 요청 상태 흐름 설계
Docker 기반 개발 환경 구성

Problem Solving

주소 검색의 한계를 보완한 지역 기반 탐색 설계
Situation
사용자가 가까운 지역의 굿즈를 찾을 수 있어야 하지만 단순 주소 문자열 검색만으로는 거리 기준 정렬과 필터링이 어려웠습니다.
Task
주소 텍스트가 아니라 사용자 좌표와 굿즈 위치를 기준으로 반경 검색과 거리 정렬이 가능한 구조가 필요했습니다.
Action
굿즈 위치를 공간 데이터로 저장하고 PostGIS 기반 반경 검색과 거리 정렬을 적용하는 방향으로 설계했습니다.
Result
현재 구현 중이며, 지역 기반 탐색과 교환 요청 흐름을 분리해 확장 가능한 구조로 정리하고 있습니다.
