병목 현상
낙관적 락(Optimistic Lock) 사용 시 트랜잭션 충돌로 인한 데드락 폭증, DB 부하로 서비스 전체 지연 현상 발생.
해결 전략
Lettuce 기반 Redis 분산 락(Spin Lock) 구조를 선적용하여 비즈니스 데이터 무결성을 유지하고, RabbitMQ 비동기 대기열로 부하를 안정 분산.
개선 결과
초당 쿠폰 발급 지연(API Latency) 42ms → 1.2ms로 97% 감소, 10,000 TPS 동시 테스트 시 초과 발급률 0% 달성.
